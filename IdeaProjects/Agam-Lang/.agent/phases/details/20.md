# Phase 20: Tree-Sitter and Prompt Engineering

## Status

- Overall status: open

## Goal

- Improve model reliability when generating Agam code.

## Main Motive

- Prevent agent systems from hallucinating Python or generic syntax when they are supposed to produce Agam.

## Responsible Crates

- `agam_parser`
- `agam_doc`
- `agam_lsp`

## Planned Work

- Write a formal Tree-Sitter grammar for Agam to constrain token generation.
- Add context-injection system prompts such as "You are an expert in Agam. Do not use Python."
- Build a few-shot context serializer that can automatically inject 5-10 high-quality Agam syntax examples, especially for ML-heavy syntax such as `tensor` and `grad`.

## What To Build Next

- Ship the Tree-Sitter grammar.
- Formalize the anti-Python prompt guidance.
- Build the reusable few-shot example injector.

## Dependencies And Prerequisites

- enough stable syntax examples and parser shape
- agent-execution surfaces from Phases 18 and 19

## Agent Interpretation

- This is one of the key anti-hallucination phases for agentic Agam use.
