# Phase 17A: Workspace Contract and Dependency Manifests

## Status

- Overall status: complete
- Current note: `agam_pkg` now carries the first source-workspace, dependency, lockfile, and environment data models, and the shared workspace session/snapshot contract now expands explicit workspace members through CLI, LSP, and daemon-facing surfaces

## Goal

- Define one stable `agam.toml` contract for package metadata, workspace layout, dependency declarations, and toolchain expectations.
- Separate source-package metadata from portable runtime package artifacts and SDK distribution manifests.

## Main Motive

- Stop each tool from inventing its own package and workspace assumptions.
- Make package identity, dependency declaration, and project layout deterministic before a resolver or registry is built on top.

## Responsible Crates

- `agam_pkg`
- `agam_driver`
- `agam_lsp`
- `agam_fmt`
- `agam_test`

## Planned Work

- Define the first stable source-package manifest schema for `agam.toml`.
- Add package identity, version, workspace members, dependency sections, target intent, and toolchain requirement fields.
- Keep the manifest aligned with the existing first-party layout from Phase 15G instead of creating a second project model.
- Define diagnostics and validation rules so malformed manifests fail clearly through first-party tooling.

## Implemented Slices

- Added `WorkspaceManifest`, `DependencySpec`, `ToolchainRequirement`, `EnvironmentSpec`, and `WorkspaceLockfile` in `agam_pkg`.
- Added TOML read/write helpers plus round-trip tests for manifest and lockfile compatibility, including the current `[project]` manifest shape.
- Switched `agamc new` to scaffold `agam.toml` through `agam_pkg::scaffold_workspace_manifest` and `agam_pkg::write_workspace_manifest_to_path`.
- Switched workspace resolution in `agam_driver` to parse `agam.toml`, honor `project.entry`, expose the manifest project name, and reject entry paths that escape the workspace root.
- Added shared `agam_pkg::expand_agam_inputs` plus crate-owned formatter and test path workflows so `agamc fmt` and `agamc test` now reuse the same manifest-root and workspace-file expansion contract instead of inventing their own file discovery rules.
- Added central `agam_pkg` manifest validation for dependency source selectors, environment metadata, and workspace-member declarations, plus parsed `WorkspaceSession` reuse so LSP and later daemon work can consume the same validated manifest/layout state instead of reparsing ad hoc.
- Wired `agam_lsp initialize` through the shared `WorkspaceSession` contract and added formatter-backed `textDocument/formatting`, so editor workflows now consume the same manifest-root and project-layout contract used by CLI tooling.
- Formalized the V1 stable manifest contract via `ManifestCompatibility` enum and `check_manifest_compatibility()`.
- Implemented `resolve_workspace_members` plus member-aware `WorkspaceSession` and `WorkspaceSnapshot` aggregation in `agam_pkg`, so explicit workspace members now contribute their source/test files and manifest fingerprints to formatter/test/LSP/daemon workflows instead of being validated but ignored.
- Updated daemon manifest invalidation and LSP workspace metadata to treat member manifests as first-class parts of the shared workspace contract, so member-manifest edits clear stale warm state and editor clients can see workspace member counts.
- Added direct local path-dependency metadata to `WorkspaceSession` and `WorkspaceSnapshot` manifest tracking, so first-party tooling can carry validated dependency-manifest state through the shared contract instead of rediscovering those manifests ad hoc later.
- Reused that direct path-dependency session metadata in top-level transitive dependency resolution, so resolver behavior no longer depends on rereading the same direct dependency manifest after the workspace session is already built.
- Extended that path-dependency metadata recursively, so nested local dependency manifests also stay attached to `WorkspaceSession` and `WorkspaceSnapshot` tracking instead of falling back to ad hoc rereads for deeper transitive graphs.
