# Phase 28: Premium Ergonomics and Syntax Cohesion

## Status

- Overall status: open

## Goal

- Make everyday Agam syntax and APIs feel premium, coherent, and low-ceremony.

## Main Motive

- Make Agam feel easy like Python while staying organized like serious systems languages.
- Align parser, formatter, and LSP around one canonical ergonomic syntax profile.

## Responsible Crates

- `agam_parser`
- `agam_ast`
- `agam_fmt`
- `agam_lsp`
- `agam_hir`
- `agam_sema`

## Planned Work

- Add named arguments and default parameter values so high-arity calls stay readable without overload explosion.
- Reduce declaration ceremony further where safety does not depend on the extra syntax.
- Unify constructor, property, and method organization across script-style code and object-style APIs.
- Feed those ergonomics rules back into `agam_fmt` and `agam_lsp` so the language not only parses the syntax, but teaches and preserves the premium style.

## What To Build Next

- Define the canonical ergonomic syntax profile.
- Implement named/default arguments and lower-ceremony syntax in a way formatter and LSP can preserve.

## Dependencies And Prerequisites

- parser and sema support for the new surface
- formatter and LSP alignment

## Agent Interpretation

- This is a language-polish phase and should reinforce Agam's identity instead of copying other languages blindly.
