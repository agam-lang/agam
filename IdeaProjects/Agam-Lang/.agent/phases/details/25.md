# Phase 25: Machine-Code Control and Optimizations

## Status

- Overall status: open

## Goal

- Expose lower-level control surfaces for systems and performance work.

## Main Motive

- Give systems and kernel-class users exact machine-level control where Agam needs to compete seriously.
- Turn existing PGO plumbing into a full guided optimization loop rather than a partial flag surface.

## Responsible Crates

- `agam_parser`
- `agam_driver`

## Planned Work

- Add inline assembly parsing and support such as `asm! { "mov eax, 1" }` for direct silicon and kernel-facing control.
- Extend the existing `--pgo-generate` and `--pgo-use` plumbing into a full production-quality PGO loop that can collect, merge, and reuse profiles.

## What To Build Next

- Complete the PGO loop into a real workflow.
- Add inline assembly only with strong syntax and safety boundaries.

## Dependencies And Prerequisites

- LLVM backend maturity
- existing PGO flag plumbing

## Agent Interpretation

- This phase should stay tightly aligned with safety and proof quality rather than becoming a raw-escape hatch everywhere.
