# Phase 17D: Environments and SDK Linking

## Status

- Overall status: completed

## Goal

- Build named Agam environments that pin compiler version, SDK packs, target packs, runtime expectations, and dependencies together.
- Make per-project environments reproducible without turning Agam into a clone of Conda.

## Main Motive

- Users need a clean way to switch between host-native, Android, data-science, and release environments without manually wiring toolchains each time.
- Environment state should be explicit project metadata, not hidden shell-global mutation.

## Responsible Crates

- `agam_pkg`
- `agam_driver`
- `agam_runtime`

## Planned Work

- Define named environments such as `dev`, `test`, `release`, and target-specific profiles.
- Bind environments to compiler versions, SDK packs, target triples, runtime backends, and dependency locks.
- Keep environment activation and inspection first-party through Agam tooling.
- Align environment metadata with `agamc doctor`, SDK discovery, and future background prewarm work.

## Checklist Status

- [x] Add a resolved-environment data model plus explicit default-selection rules on top of `agam.toml` + `agam.lock`.
- [x] Add `agamc env list` and `agamc env inspect` for first-party environment inspection.
- [x] Thread `--env` selection through build/run/dev/doctor/package flows.
- [x] Add project-local selection/diagnose flows without falling back to shell-global mutation.

## Implemented Slices

- Added `ResolvedEnvironment`, `default_environment_name()`, `select_environment_name()`, `resolve_environment_catalog()`, and `resolve_environment()` in `agam_pkg` so environment selection is deterministic and shared instead of being reimplemented ad hoc in tooling.
- Added `agamc env list` and `agamc env inspect` in `agam_driver`, resolving environments against the current manifest and an in-memory lockfile so the CLI can show compiler/sdk/target/backend/package bindings without mutating the workspace.
- Added focused package and driver tests covering default selection, ambiguous-environment rejection, merged toolchain fallback, and the new CLI inspection path.
- Threaded environment resolution into `agamc build`, `run`, and `dev` so the selected environment can now supply default target/backend settings while explicit CLI flags still win.
- Extended `agamc doctor` with optional workspace-rooted environment diagnostics and `agamc package sdk` with environment-aware target metadata plus manifest notes, keeping selection explicit and project-local instead of shell-global.

## What To Build Next

- Keep the environment contract explicit and project-local as higher-level tooling grows around it.
- Make SDK and target-pack linkage reproducible before adding shell integration or convenience shortcuts.
- Keep environments layered on top of manifests and lockfiles rather than inventing a second resolver.

## Remaining Work

- Shell-global activation remains intentionally out of scope; environment choice stays explicit through `agam.toml` plus per-command selection.
- Future work can add richer target-pack packaging and hosted environment distribution without changing the current explicit selection contract.

## Dependencies And Prerequisites

- manifest contract from 17A
- lockfile contract from 17B
- registry metadata from 17C
- SDK contract from 15H

## Agent Interpretation

- This is an environment-management phase, not a system-package-manager phase.
- Interoperate with foreign toolchains, but do not absorb foreign package managers into Agam's core contract.
