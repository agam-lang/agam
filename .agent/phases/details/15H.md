# Phase 15H: Native LLVM SDK Distribution and Toolchain Bundles

## Status

- Overall status: completed

## Goal

- Ship native LLVM SDK outputs that keep `agamc`, bundled LLVM, target metadata, and first-party readiness checks under one supportable contract.
- Make Windows and Linux SDK distribution real, repeatable, and CI-verifiable before expanding target-pack coverage.

## Main Motive

- Keep native LLVM support installable and diagnosable as a first-party product instead of a loose set of local setup steps.
- Separate the small compiler frontend from large host and target toolchain payloads without making readiness opaque.

## Responsible Crates

- `agam_driver`
- `agam_pkg`
- `agam_runtime`

## Implemented Slices

- Added `agamc package sdk` as the first host-native SDK staging workflow.
- Added the SDK distribution manifest contract in `agam_pkg` so packaged compiler bundles can describe LLVM layout, compiler binary location, and supported targets explicitly.
- Added bundled LLVM layout support so `agamc`, packaged SDKs, and first-party readiness checks target the same filesystem contract.
- Added `agamc doctor` and aligned it with the native LLVM readiness story instead of relying on undocumented host setup assumptions.
- Added the initial `.github/workflows/sdk-dist.yml` workflow skeleton plus `scripts/package_sdk.py` support for staged distribution assembly.
- Restored `scripts/package_sdk.py` and `.github/workflows/sdk-dist.yml` as real host-validation surfaces that build `agamc`, seed a minimal bundled-LLVM layout from runner toolchains, run `agamc package sdk`, and validate the emitted SDK manifest plus filesystem contract on Windows and Linux hosts.
- Extended `scripts/package_sdk.py` to emit release-ready `.zip` or `.tar.gz` SDK archives plus `.sha256` checksum files from the same validated staging directory used locally and in CI.
- Replaced the old reusable-workflow stub with a self-contained Windows/Linux matrix pipeline that builds, validates, archives, uploads SDK artifacts on every run, and publishes those assets on GitHub releases.
- Added packaged Android target-pack metadata to `SdkDistributionManifest` and taught `agamc package sdk --android-sysroot …` to stage `target-packs/android-arm64/sysroot` into the SDK layout.
- Taught LLVM Android target resolution and `agamc doctor` to consume packaged SDK target packs when present, so installed SDKs can self-describe Android readiness instead of relying only on host-global environment variables.
- Extended `scripts/package_sdk.py` with Android sysroot discovery, `--require-android-target-pack`, and packaged-target-pack validation so local and CI packaging use the same contract.
- Added Linux hosted-runner Android NDK provisioning plus a follow-up artifact verification job in `.github/workflows/sdk-dist.yml` that downloads SDK archives, checks their `.sha256` files, extracts them, and re-validates the manifest/layout contract after upload.
- Hardened `sdk-dist.yml` for real GitHub hosted runners: added `working-directory: agam` at job level, fixed artifact upload paths to resolve relative to `GITHUB_WORKSPACE`, added `actions/checkout@v4` in the publish-release job for proper `gh` CLI repo context, added Python 3.12 `tarfile.data_filter` security, improved CI log output with manifest summaries and asset inspection, and separated the compiler build step from SDK packaging for clearer failure isolation.
- Extended `scripts/package_sdk.py` with `--release` for optimized distribution builds and `--skip-build` to reuse pre-built compiler binaries in CI, added `clang-cl.exe` companion detection on Windows runners, and added post-validation manifest summary output for CI log visibility.
- Added `scripts/validate_sdk_e2e.py` as a local end-to-end validation script that mirrors the full CI pipeline (build → package → archive → extract → checksum → manifest validation) so the SDK contract can be proven locally without hosted runners.
