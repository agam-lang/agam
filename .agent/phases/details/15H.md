# Phase 15H: Native LLVM SDK Distribution and Toolchain Bundles

## Status

- Overall status: partial
- Why it matters now: Agam already has SDK packaging and readiness checks, but the hosted artifact and target-pack validation story is not finished

## Goal

- Ship native LLVM SDK outputs that keep `agamc`, bundled LLVM, target metadata, and first-party readiness checks under one supportable contract.
- Make Windows and Linux SDK distribution real, repeatable, and CI-verifiable before expanding target-pack coverage.

## Main Motive

- Keep native LLVM support installable and diagnosable as a first-party product instead of a loose set of local setup steps.
- Separate the small compiler frontend from large host and target toolchain payloads without making readiness opaque.

## Responsible Crates

- `agam_driver`
- `agam_pkg`
- `agam_runtime`

## Planned Work

- Validate hosted-runner SDK builds for Windows and Linux using the bundled LLVM layout.
- Publish real host SDK artifacts instead of only local staging output.
- Add Android target-pack packaging and validation on top of the host SDK flow.
- Keep `agamc doctor` and SDK manifests aligned with the real toolchain/readiness contract.

## Implemented Slices

- Added `agamc package sdk` as the first host-native SDK staging workflow.
- Added the SDK distribution manifest contract in `agam_pkg` so packaged compiler bundles can describe LLVM layout, compiler binary location, and supported targets explicitly.
- Added bundled LLVM layout support so `agamc`, packaged SDKs, and first-party readiness checks target the same filesystem contract.
- Added `agamc doctor` and aligned it with the native LLVM readiness story instead of relying on undocumented host setup assumptions.
- Added the initial `.github/workflows/sdk-dist.yml` workflow skeleton plus `scripts/package_sdk.py` support for staged distribution assembly.

## Remaining Work

- Hosted-runner validation and actual artifact publication are still open.
- Android target-pack packaging and validation still need to be layered onto the host SDK contract.

## Strategic Next Step

- Verify that CI-produced SDK outputs match the contract already assumed by `agamc doctor` and local SDK packaging.

## What To Build Next

- Run the Windows and Linux SDK flow on hosted runners with bundled LLVM in the loop.
- Add validation that the emitted SDK manifests, compiler binary, and LLVM bundle layout agree.
- Extend the same contract with Android sysroot or target-pack validation after the host SDK flow is stable.

## Agent Interpretation

- Treat 15H as a supportability phase, not just a packaging script phase.
- Keep `agamc doctor`, SDK manifests, bundled LLVM layout, and published artifacts on one contract instead of letting CI and local workflows drift.
