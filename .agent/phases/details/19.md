# Phase 19: LangChain and LlamaIndex Wrappers

## Status

- Overall status: partial

## Goal

- Make Agam easy to adopt in popular AI-agent ecosystems.

## Main Motive

- Reduce adoption friction by making Agam feel like a drop-in execution upgrade for existing Python-based agent stacks.

## Responsible Crates

- `agam_ffi`
- `agam_runtime`
- `agam_pkg`

## Planned Work

- Build `AgamREPLTool` as a native Python class.
- Publish it as a drop-in replacement for `PythonREPLTool` in LangChain and LlamaIndex.
- Extend `agam_ffi` so existing Python frameworks can seamlessly call compiled native Agam.

## Implemented Slices

- Added `AgamExecClient` in `agam_ffi` as a thin subprocess bridge over `agamc exec --json`, so external integrations no longer need to know the CLI details themselves.
- Added `AgamReplTool` in `agam_ffi` as the first reusable tool abstraction for configured Agam execution requests, including backend, filename, optimization level, fast mode, and runtime arguments.
- Added crate-level tests for request construction plus structured-response parsing through a mock execution tool, so the wrapper foundation is exercised without depending on a live Python binding yet.
- Added `crates/agam_ffi/python` as the first Python package scaffold, including Python-native `HeadlessExecutionRequest`, `HeadlessExecutionResponse`, `AgamExecClient`, and `AgamREPLTool` wrappers over the same `agamc exec --json` contract plus stdlib-only unit tests.
- Added optional Python package extras plus `AgamREPLTool.to_langchain_structured_tool()` and `AgamREPLTool.to_llamaindex_tool()` adapter hooks, so the first packaged LangChain/LlamaIndex integration story now sits on top of the same strict `agamc exec --json` request contract.
- Validated those adapter hooks against live `langchain-core` and `llama-index-core` installs through `uv run` smoke tests, so the current wrapper shape now matches real upstream framework APIs instead of only repo-local stubs.
- Added publish-ready package metadata plus a dedicated `.github/workflows/agam-ffi-python.yml` workflow that runs the Python wrapper tests, builds `sdist`/wheel artifacts, validates them with `twine check`, and publishes them on GitHub releases through the standard PyPI action.

## Remaining Work

- The adapter hooks, packaging extras, and release workflow now exist in-repo, but one external package release still needs to be exercised end to end and kept healthy as upstream framework APIs evolve.

## What To Build Next
