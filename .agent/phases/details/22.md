# Phase 22: Omni-Targeting Directives

## Status

- Overall status: completed

## Goal

- Make Agam target radically different deployment environments through language-level target directives.

## Main Motive

- Make Agam truly ubiquitous instead of a desktop-only or server-only language.
- Scale from tiny bare-metal deployments to heavy enterprise and HPC targets through explicit target semantics.

## Responsible Crates

- `agam_sema`
- `agam_codegen`
- `agam_hir`
- `agam_mir`

## Completed Work

- `agam_sema::target` module with `TargetProfile` enum (`Default`, `Iot`, `Enterprise`, `Hpc`), `TargetConstraints`, `resolve_target_profile()`, and `validate_effect_for_target()`
- Parser already supported `@name.subname` annotations — `@target.iot`, `@target.enterprise`, `@target.hpc` parse without any parser changes
- `HirFunction` now carries `target: TargetProfile` resolved from function annotations
- `MirFunction` now carries `target: TargetProfile` propagated from HIR (serde-defaulted for backwards compat)
- C emitter conditionally skips effect/dataframe/tensor preludes for IoT modules; emits `AGAM_NO_HEAP` and `AGAM_HPC` defines
- LLVM emitter emits function-level target comments and module-level `!agam.target.*` named metadata nodes
- HIR lowering rejects `perform` statements in `@target.iot` functions with diagnostic errors
- 12 sema unit tests for profile resolution, constraint derivation, and validation
- 5 HIR unit tests for target propagation and IoT effect rejection

## Dependencies And Prerequisites

- mature LLVM and SDK story
- clear runtime contract and target-pack direction

## Agent Interpretation

- This phase should build on the current LLVM and SDK platform work, not run ahead of it.
