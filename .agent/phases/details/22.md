# Phase 22: Omni-Targeting Directives

## Status

- Overall status: open

## Goal

- Make Agam target radically different deployment environments through language-level target directives.

## Main Motive

- Make Agam truly ubiquitous instead of a desktop-only or server-only language.
- Scale from tiny bare-metal deployments to heavy enterprise and HPC targets through explicit target semantics.

## Responsible Crates

- `agam_sema`
- `agam_codegen`

## Planned Work

- Implement `@target.iot` so the compiler can strip `agam_std`, strip ARC memory, refuse heap allocations, and target sub-10KB static binaries for devices such as ESP32 and Arduino.
- Implement `@target.enterprise` to enable heavier runtime services such as GC choice, thread pooling, and capability-sandbox locking.
- Implement `@target.hpc` so `agam_codegen` and `agam_mir` can prefer aggressive vectorization and compute-oriented behavior.

## What To Build Next

- Design the semantic meaning of the target directives.
- Then wire the target-specific runtime/codegen restrictions and preferences.

## Dependencies And Prerequisites

- mature LLVM and SDK story
- clear runtime contract and target-pack direction

## Agent Interpretation

- This phase should build on the current LLVM and SDK platform work, not run ahead of it.
