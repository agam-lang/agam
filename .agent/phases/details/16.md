# Phase 16: Interactive REPL and Sandboxed Execution

## Status

- Overall status: complete
- Why it matters now: once the daemon/workspace and SDK slices stabilize, Agam needs a first-party interactive execution surface instead of forcing a full build loop for every experiment

## Goal

- Build an interactive REPL for Agam.
- Build a headless sandboxed execution surface that accepts Agam strings and returns structured results.

## Main Motive

- Let users and agent systems evaluate Agam interactively without forcing a full build/package cycle every time.
- Provide a clean execution surface for notebooks, exploration, and future LLM-facing runtime services.

## Responsible Crates

- `agam_driver`
- `agam_notebook`
- `agam_jit`
- `agam_runtime`

## Planned Work

- Build the core REPL loop on top of `agam_jit` and the existing `agamc repl` CLI surface.
- Define a strict JSON contract for headless execution requests plus `stdout`, `stderr`, and result reporting.
- Keep the execution surface isolated and machine-consumable enough for notebook and tool-driven flows.
- Reuse the stable workspace/daemon direction from 15F and 15G instead of inventing a second execution path.

## Implemented Slices

- Added strict `agam_notebook::{HeadlessExecutionRequest, HeadlessExecutionResponse}` JSON contracts with unknown-field rejection, explicit backend selection, filename normalization, and structured exit/stdout/stderr/error reporting.
- Replaced the old `agamc repl` placeholder with a real buffered shell that supports `:help`, `:run`, `:show`, `:reset`, `:load`, `:backend`, `:opt`, `:fast`, and `:quit`.
- Added `agamc repl --json` so one Agam source request can be read from stdin and answered with one machine-readable JSON response instead of CLI-string scraping.
- Switched interactive `:run` execution onto the in-process run path used by the main CLI instead of shelling back through `agamc run`, so REPL sessions now exercise the shared backend/JIT flow directly.
- Added a REPL-owned temporary execution cache that materializes the current snippet once and reuses cached warm lowered state across repeated `:run` calls when the buffered source has not changed.
- Added buffered diagnostic capture in `agam_errors` plus captured JIT runtime stdout in `agam_jit`, so headless execution no longer has to scrape terminal output for the JIT path.
- Switched JIT-backed `agamc repl --json` requests onto an in-process parse/resolve/type-check/lower/JIT path that returns structured `stdout`, buffered `stderr`, and exit status without shelling back through `agamc run`.
- Replaced the remaining non-JIT headless subprocess bridge with direct in-process LLVM/C execution, so `agamc repl --json` now materializes one temporary source file, compiles from the already-built warm MIR, and captures native `stdout`/`stderr` without shelling back through the CLI.
- Replaced the old unchanged-buffer-only reuse with a REPL-owned `DaemonSession` + `refresh_daemon_session()` path, so buffer edits now stay on the same incremental warm-state contract used by the shared daemon pipeline.
