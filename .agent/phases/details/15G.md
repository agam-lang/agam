- Overall status: partial
- Why it matters now: several visible workflow surfaces are already shipped, but the unification work is still open

## Goal

- Make Agam tooling feel like one coherent first-party product rather than a set of unrelated commands.

## Implemented Slices

- Shipped `agamc doctor` as the first native backend and SDK readiness command.
- Added `agamc new`, `agamc dev`, and `agamc cache status`.
- Defined the first explicit first-party workspace contract around `agam.toml`, `src/main.agam`, `tests/*.agam`, and `.agam_cache/`.
- Added a structured benchmark workspace under `benchmarks/` with categorized suites, harnesses, CI helpers, result roots, and methodology docs, while keeping `.agent/test/` as the localized phase-work microbenchmark area.
- Expanded the benchmark workspace with explicit Win11/Linux/WSL environment profiles, Agam LLVM/JIT/C target variants, call-cache on/off comparisons, Clang++/Clang/Rust/CPython/Go comparison targets, and richer space-profile reporting for SSD, RAM, cache capacity, and register-budget context.
- Tightened benchmark fairness by running Agam through the built driver binary instead of `cargo run`, compiling emitted C backend output to a native executable before runtime measurement, and adding a compile warmup pass before measured AOT compile timing.
- Published the first same-host README benchmark snapshot and plot set for the `01_algorithms/fibonacci` comparison matrix so public docs show a concrete Win11 run instead of just the benchmark scaffold.
- Added a reproducible benchmark scorecard output plus README winner summary so the public benchmark section can rank targets and explain the current snapshot without hiding the underlying raw metrics.
- Restored the shared package/runtime contract under `agam_runtime::contract` and re-enabled serde-backed MIR/package serialization so portable package metadata, runtime ABI data, and workspace manifest defaults compile and round-trip under `agam_pkg` again.
- Restored the repo-local `.agam_cache` runtime surface and a conservative MIR escape/promotion entrypoint so `agamc build`, `agamc package pack`, `agamc cache status`, and `agamc dev` compile against the same cache/pipeline contract again while deeper optimization work stays staged separately.
- Replaced the empty `agam_test` and `agam_lsp` stubs with a real file-based JIT test runner and a minimal stdio JSON-RPC loop so the first-party tool crates expose concrete entrypoints instead of dangling driver hooks.
- Moved workspace layout discovery into `agam_pkg` and kept `agam_driver` on thin wrappers so the manifest-root, entry-file, source-file, and test-file contract now lives in the package/tooling layer where formatter, tests, LSP, and later daemon work can reuse it.
