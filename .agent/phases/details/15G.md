# Phase 15G: First-Party Premium Experience Layer

## Status

- Overall status: completed
- Why it matters now: the first-party tooling surfaces now share one workspace contract instead of drifting independently

## Goal

- Make Agam tooling feel like one coherent first-party product rather than a set of unrelated commands.
- Unify workspace conventions, manifests, cache metadata, package/runtime metadata, tooling, and the future daemon behind one stable contract.

## Main Motive

- Make Agam feel like one premium first-party environment instead of many loosely coordinated tools.
- Keep manifests, cache layout, diagnostics, package/runtime metadata, and editor/test workflows under one contract.

## Implemented Slices

- Shipped `agamc doctor` as the first native backend and SDK readiness command.
- Added `agamc new`, `agamc dev`, and `agamc cache status`.
- Defined the first explicit first-party workspace contract around `agam.toml`, `src/main.agam`, `tests/*.agam`, and `.agam_cache/`.
- Added a structured benchmark workspace under `benchmarks/` with categorized suites, harnesses, CI helpers, result roots, and methodology docs, while keeping `.agent/test/` as the localized phase-work microbenchmark area.
- Expanded the benchmark workspace with explicit Win11/Linux/WSL environment profiles, Agam LLVM/JIT/C target variants, call-cache on/off comparisons, Clang++/Clang/Rust/CPython/Go comparison targets, and richer space-profile reporting for SSD, RAM, cache capacity, and register-budget context.
- Expanded the benchmark catalog itself to 38 Agam workloads, including additional algorithm, numerical, data-structure, string, and I/O cases plus dedicated call-cache hot-set, mixed-locality, shifting-working-set, and unique-input benchmarks so call-cache evaluation is no longer effectively represented by Fibonacci alone.
- Tightened benchmark fairness by running Agam through the built driver binary instead of `cargo run`, compiling emitted C backend output to a native executable before runtime measurement, and adding a compile warmup pass before measured AOT compile timing.
- Published the first same-host README benchmark snapshot and plot set for the `01_algorithms/fibonacci` comparison matrix so public docs show a concrete Win11 run instead of just the benchmark scaffold.
- Added a reproducible benchmark scorecard output plus README winner summary so the public benchmark section can rank targets and explain the current snapshot without hiding the underlying raw metrics.
- Added `benchmarks/results/README.md` so the checked-in result root now documents which raw run backs the published same-host snapshot, which later dry-run validates the newly added workloads, and why the latest aggregated CSVs can legitimately be compile-only.
- Added workload-by-workload result-status tables to `benchmarks/results/README.md` so the benchmark docs now state explicitly which comparison-ready cases already have a checked-in measured snapshot, which only have checked-in dry-run validation, and which are still source-only; the same result index now also records the four Agam-only call-cache locality shapes separately instead of leaving their status implicit.
- Added `benchmarks/COVERAGE_MATRIX.md` so the benchmark docs now track a 130-workload implemented-vs-future map instead of over-implying that the public Fibonacci snapshot is the whole benchmark surface; the matrix now records not just the current CPU-first slice but also the broader methodology backlog across CPU, memory, storage, network, accelerator, interconnect, and quantum-simulator workloads, while still separating comparison-ready cases from planned and future-lab entries.
- Expanded the comparison-language coverage itself with C/C++/Go/Rust/Python counterparts for `edit_distance`, `polynomial_eval`, `ring_buffer`, `token_frequency`, and `csv_scanning`, lifting the benchmark workspace to 38 checked-in comparison sources and 8 comparison-ready workload families instead of leaving Fibonacci, matrix multiply, and tensor matmul as the only practical cross-language anchors.
- Restored the shared package/runtime contract under `agam_runtime::contract` and re-enabled serde-backed MIR/package serialization so portable package metadata, runtime ABI data, and workspace manifest defaults compile and round-trip under `agam_pkg` again.
- Restored the repo-local `.agam_cache` runtime surface and a conservative MIR escape/promotion entrypoint so `agamc build`, `agamc package pack`, `agamc cache status`, and `agamc dev` compile against the same cache/pipeline contract again while deeper optimization work stays staged separately.
- Replaced the empty `agam_test` and `agam_lsp` stubs with a real file-based JIT test runner and a minimal stdio JSON-RPC loop so the first-party tool crates expose concrete entrypoints instead of dangling driver hooks.
- Moved workspace layout discovery into `agam_pkg` and kept `agam_driver` on thin wrappers so the manifest-root, entry-file, source-file, and test-file contract now lives in the package/tooling layer where formatter, tests, LSP, and later daemon work can reuse it.
- Moved input expansion into `agam_pkg` and path-based workflows into `agam_fmt` and `agam_test`, so `agamc fmt`, `agamc test`, and `agamc dev` now consume the same workspace-aware source/test file selection contract instead of carrying separate recursive file walkers in the driver.
- Routed `agamc check` through that same shared `agam_pkg::expand_agam_inputs` contract, so workspace roots and manifest-based paths resolve to the same concrete source/test file set already used by formatter/test/dev workflows instead of behaving like raw one-file-only inputs.
- Routed `agamc build`, `agamc run`, and `agamc package pack` through the shared workspace entry-file resolver too, so those single-entry product surfaces now accept a workspace root or `agam.toml` path and converge on the same configured entry file instead of forcing callers to spell out `src/main.agam`.
- Kept `agamc build --output …` on that same entry-file contract so a single workspace-root or manifest input still resolves to the configured entry file before output-path selection instead of regressing back to treating the directory itself as source input.
- Made the shared build path create missing parent directories for explicit outputs before cache restore or backend emission, so `agamc build --output nested/dir/program.exe` behaves like a first-party workflow instead of failing on the first missing folder.
- Deduplicated overlapping `agamc build` workspace-root, manifest, and source-file inputs after entry resolution so the first-party build queue stays deterministic without recompiling the same configured entry file multiple times from aliased paths.
- Moved default manifest-backed `agamc build` artifacts into `dist/<project>` or `dist/<project>.exe` while keeping single-file builds sibling to the source, so native outputs follow the same first-party artifact area instead of landing in `src/`.
- Reused that same workspace-aware output path for native host `agamc run` staging too, so non-JIT run/dev flows stop dropping temporary C/LLVM/native artifacts into `src/` for manifest-backed projects.
- Moved default `agamc package pack` output for manifest-backed workspaces into `dist/<project>.agpkg.json` while keeping single-file scripts sibling to their source, so portable package artifacts follow the first-party artifact area instead of polluting `src/`.
- Removed the stale `src/main` ignore from newly scaffolded workspace `.gitignore` files now that first-party build artifacts default to `dist/`, keeping the project template aligned with the actual artifact contract.
- Added validated `agam_pkg` workspace sessions plus formatter-backed LSP document formatting, so `agam_lsp initialize` now resolves the same parsed manifest/layout contract used by CLI workflows and malformed manifest metadata fails consistently across editor and command surfaces.
- Built the 15F daemon entirely on top of `WorkspaceSession` + `WorkspaceSnapshot`, completing the unification item instead of introducing a parallel daemon-only project model.
