# Phase 15F: Incremental Daemon, Background Prewarm, and Parallel Compilation

## Status

- Overall status: open
- Why it matters now: the native LLVM path is already deep enough that reparsing and relowering everything on each edit is now a real product drag

## Goal

- Keep parsed, typed, and lowered state warm across edits instead of rebuilding the full pipeline every time.
- Add deterministic parallel compilation and background prewarm without sacrificing diagnostic stability.

## Main Motive

- Make the first-party Agam workflow feel immediate on real projects instead of recompiling cold on every command.
- Prepare the CLI, LSP, and future cache/package surfaces to reuse one long-lived workspace pipeline instead of separate one-shot runs.

## Responsible Crates

- `agam_driver`
- `agam_lsp`
- `agam_pkg`
- `agam_parser`
- `agam_sema`
- `agam_hir`
- `agam_mir`
- `agam_codegen`
- `agam_jit`

## Planned Work

- Add a persistent daemon that owns workspace discovery plus parsed, typed, and lowered state.
- Define an incremental invalidation model so one file edit does not throw away unrelated work.
- Add deterministic parallel scheduling for independent package, module, and backend work.
- Prewarm common compilation products in the background so later CLI and editor requests hit a warm pipeline.
- Keep diagnostics, spans, and cache keys stable even when work is reused or parallelized.
