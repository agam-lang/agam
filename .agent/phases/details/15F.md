# Phase 15F: Incremental Daemon, Background Prewarm, and Parallel Compilation

## Status

- Overall status: partial
- Why it matters now: the native LLVM path is already deep enough that reparsing and relowering everything on each edit is now a real product drag

## Goal

- Keep parsed, typed, and lowered state warm across edits instead of rebuilding the full pipeline every time.
- Add deterministic parallel compilation and background prewarm without sacrificing diagnostic stability.

## Main Motive

- Make the first-party Agam workflow feel immediate on real projects instead of recompiling cold on every command.
- Prepare the CLI, LSP, and future cache/package surfaces to reuse one long-lived workspace pipeline instead of separate one-shot runs.

## Responsible Crates

- `agam_driver`
- `agam_lsp`
- `agam_pkg`
- `agam_parser`
- `agam_sema`
- `agam_hir`
- `agam_mir`
- `agam_codegen`
- `agam_jit`

## Planned Work

- Add a persistent daemon that owns workspace discovery plus parsed, typed, and lowered state.
- Define an incremental invalidation model so one file edit does not throw away unrelated work.
- Add deterministic parallel scheduling for independent package, module, and backend work.
- Prewarm common compilation products in the background so later CLI and editor requests hit a warm pipeline.
- Keep diagnostics, spans, and cache keys stable even when work is reused or parallelized.

## Checklist Status

- [ ] Add a persistent daemon process that keeps parsed, typed, and lowered state warm across edits.
- [ ] Add deterministic incremental invalidation so unrelated files do not get recompiled unnecessarily.
- [ ] Add deterministic parallel compilation for independent work without reordering user-facing diagnostics.
- [ ] Add background prewarm hooks that prepare hot CLI and editor workflows without hiding failures.

## Implemented Slices

- Added `agam_pkg::WorkspaceSnapshot`, `WorkspaceFileSnapshot`, and `WorkspaceSnapshotDiff` so the shared workspace contract can fingerprint manifest, source, and test files and report added/changed/removed paths as the first explicit invalidation surface for later daemon work.

## Remaining Work

- The full daemon, warm-state retention, background prewarm, and parallel-scheduling story is still open.
- The new snapshot/diff contract still needs to be consumed by a real long-lived daemon or editor service.

## Strategic Next Step

- Build daemon-owned warm parse/type/lower state on top of the new workspace snapshot and diff contract from `agam_pkg`.

## What To Build Next

- Define the first daemon-owned warm workspace session model.
- Consume the snapshot/diff contract at the parse/type/lower boundaries so one-file edits do not invalidate the whole workspace blindly.
- Start with deterministic warm-state reuse before layering in parallel execution.

## Dependencies And Prerequisites

- workspace contract reuse from 15G
- manifest stabilization from 17A
- current native LLVM and JIT pipelines

## Agent Interpretation

- Treat 15F as a correctness-sensitive systems phase, not a background-service wrapper task.
- Do not trade deterministic diagnostics or phase traceability for concurrency or cache reuse.
