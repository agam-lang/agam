# Phase 15F: Incremental Daemon, Background Prewarm, and Parallel Compilation

## Status

- Overall status: partial
- Why it matters now: the native LLVM path is already deep enough that reparsing and relowering everything on each edit is now a real product drag

## Goal

- Keep parsed, typed, and lowered state warm across edits instead of rebuilding the full pipeline every time.
- Add deterministic parallel compilation and background prewarm without sacrificing diagnostic stability.

## Main Motive

- Make the first-party Agam workflow feel immediate on real projects instead of recompiling cold on every command.
- Prepare the CLI, LSP, and future cache/package surfaces to reuse one long-lived workspace pipeline instead of separate one-shot runs.

## Planned Work

- Add a persistent daemon that owns workspace discovery plus parsed, typed, and lowered state.
- Define an incremental invalidation model so one file edit does not throw away unrelated work.
- Add deterministic parallel scheduling for independent package, module, and backend work.
- Prewarm common compilation products in the background so later CLI and editor requests hit a warm pipeline.
- Keep diagnostics, spans, and cache keys stable even when work is reused or parallelized.

## Implemented Slices

- Added `agam_pkg::WorkspaceSnapshot`, `WorkspaceFileSnapshot`, and `WorkspaceSnapshotDiff` so the shared workspace contract can fingerprint manifest, source, and test files and report added/changed/removed paths as the first explicit invalidation surface for later daemon work.
- Added `WorkspaceSnapshot::is_stale()` check for incremental invalidation.
- Created the foundation for the persistent daemon process in `agam_driver` (`DaemonSession`, `WarmState`, and `IncrementalPipeline`).
- Added the `agamc daemon` subcommand and the initial hooks in `agamc dev` to detect actively running daemon processes.
- Landed a real foreground warm-state daemon loop in `agam_driver` that snapshots a workspace, keeps parsed/HIR/MIR state warm across refreshes, invalidates per-file state via `WorkspaceSnapshotDiff`, and clears the full warm cache when the manifest changes.
- Added persisted daemon heartbeat/status metadata under `.agam_cache/daemon/status.json` so `agamc daemon status` and `agamc dev` can report live vs stale warm-state sessions without the old placeholder env-var check.
- Made `agamc daemon --once` persist a one-shot status snapshot too, so follow-up status checks can observe the last warm refresh immediately instead of only long-running loops.
- Marked one-shot warm refreshes distinctly from live foreground daemon loops so `agamc dev` only treats a real loop as connected instead of mistaking a fresh snapshot for an active daemon.
- Split `agamc dev` daemon reporting into connected, snapshot, stale, and not-connected cases so the first-party workflow surfaces warm-state availability accurately instead of collapsing every non-live status into “stale”.
- Made daemon status parsing backward-compatible with older status files and persisted explicit daemon error state on failed warm refreshes so the latest failure is visible instead of silently leaving only the previous good snapshot behind.
- Added `agamc daemon … clear` so snapshot, stale, or error status files can be cleared explicitly as part of the foreground daemon lifecycle instead of only aging out or requiring manual cache deletion.
- Relaxed daemon status/clear path resolution so those lifecycle commands can still locate the workspace from a deleted source-path hint instead of failing before they even reach the persisted daemon metadata.
- Extended that daemon metadata lookup to existing workspace-root directories that still have persisted daemon state even after their entry file disappears, so lifecycle commands can recover from partial workspace deletion without manual path surgery.
- Refactored daemon refresh into an explicit cycle step so foreground loop mode can survive transient parse/type errors, persist error status for the bad edit, and recover automatically on the next successful refresh instead of exiting on the first invalid file version.
- Fixed foreground daemon refresh hints for manifest-free single-file sessions so later cycles keep snapshotting the actual `.agam` file instead of incorrectly switching to the workspace root and breaking recovery after the first refresh.
- Routed CLI semantic checking through the real resolver/type-checker pipeline so `agamc check` and `agamc dev` stop reporting parse-only success as full type-check success.
- Reused a warmed entry-file frontend artifact inside `agamc dev` so the final run step can consume prevalidated MIR/source-feature state instead of reparsing and relowering the entry file a second time in the same invocation.
- Kept `agamc dev --no-run` on the semantic-only path for the entry file so pure validation runs do not pay unnecessary HIR/MIR lowering cost.

## Remaining Work

- IPC-backed daemon/client coordination is still open; the current implementation is a foreground warm-state loop rather than a detached background service.
- Warm-state reuse still needs to cross process boundaries and feed more build/run/check workflows directly from a daemon-owned session instead of only within one-shot CLI execution or daemon-owned refreshes.
- Deterministic parallel scheduling and background prewarm are still open.
