# Phase 15D: Value Profiling, Adaptive Admission, and Specialization

## Status

- Overall status: complete
- Why it mattered now: `info.md` listed this as the first immediate next phase and the remaining open work was still on the critical LLVM/JIT optimization path

## Goal

- Make measured payoff drive cache admission and specialization decisions instead of fixed heuristics.
- Keep specialization guarded, reversible, and portable.

## Main Motive

- Make optimization policy follow measured runtime evidence instead of fixed heuristics or hand tuning.
- Keep specialization profitable and reversible rather than baking unstable hot-path guesses into the compiler.

## Responsible Crates

- `agam_profile`
- `agam_runtime`
- `agam_jit`
- `agam_codegen`

## Implemented Slices

- Added `agam_profile` models for adaptive admission decisions, stable scalar value summaries, reuse-distance snapshots, and specialization hints.
- Wired JIT call-cache admission to measured payoff signals instead of blindly storing every miss in basic mode.
- Extended verbose JIT reporting with unique-key counts, reuse-distance summaries, stable scalar values, and specialization hints.
- Added persisted JIT call-cache profiles so later runs can pre-promote hot functions into optimize mode.
- Added the first profile-guided specialization cloning slice on the JIT path using persisted stable scalar profiles.
- Added the first LLVM-native runtime profile export/import slice through sidecar profile files.
- Added specialization-guard hit and fallback tracking to both JIT and LLVM call-cache specialization paths.
- Extended persisted profile selection so measured poor specialization match rates suppress future clone plans until later runs produce better evidence.
- Extended persisted optimize pre-promotion and specialization retention so favorable specialization guard evidence keeps profitable clones alive while poor match rates stop specialization-only promotions.
- Fed specialization guard feedback into live JIT admission payoff and optimized victim scoring so runtime cache policy reacts to measured guarded-clone behavior during the same run.
- Mirrored specialization guard feedback weighting on the LLVM optimized cache wrapper so live LLVM admission and replacement scoring also react to measured guarded-clone behavior.
- Replaced LLVM's remaining fixed optimized-cache admission gate with an emitted adaptive payoff score based on candidate repeats, reuse distance, hit rate, stable scalar evidence, cache pressure, and specialization feedback.
- Added LLVM-side observed unique-key and hottest-key hit tracking so live admission scoring and exported profile imports stop inferring those signals from cache length or store counts.
- Tightened LLVM unique-key observation to use a bounded per-function observed-key registry instead of incrementing spread counters on every miss or pending reset.
- Switched LLVM optimize-mode admission to use candidate-local observed hit and reuse signals from that bounded registry, and fixed the emitted observed-key scan PHI predecessor so native clang accepts optimized multi-argument cache wrappers.
- Mirrored the JIT observed-key retention policy on LLVM by replacing the coldest/oldest observed slot once the bounded registry fills instead of freezing key-shape observation at the first 64 keys.
- Mirrored JIT reuse-distance accounting on LLVM by recording observed-key reuse into exported `RD` profile summaries even before a key is admitted into the live cache.
- Split LLVM stable-value profiling into vote scores plus exported candidate match counts so `SV` summaries and live stable-slot admission checks track a more faithful dominant-value frequency instead of reusing the Boyer-Moore score directly.
- Decoupled persisted guarded specialization from optimize-mode prepromotion so stable-value evidence can keep guarded clones alive on basic cache-enabled paths while optimize promotion remains tied to independent payoff signals such as hit rate, reuse distance, or favorable specialization feedback.
- Removed LLVM's remaining optimize-only specialization filter so explicit basic call-cache wrappers can also emit and run guarded specialization clones instead of silently dropping persisted specialization plans outside optimize mode.
- Added live guarded-specialization disablement on both JIT and LLVM paths so once same-run specialization feedback becomes clearly unfavorable, later calls bypass clone guards and fall straight back to the generic implementation without continuing to accumulate fallback noise.
- Refined live guarded-specialization disablement to track feedback per clone on both JIT and LLVM, so an unfavorable narrow guard can disable itself without shutting off broader guarded clones for the same function during the same run.
- Extended the shared/JIT persisted call-cache profile model to retain clone-specific guarded-specialization feedback and use that data when re-planning specializations, so a bad narrow clone no longer suppresses every broader persisted plan for the same function on later JIT runs.
- Extended LLVM call-cache profile export/import to schema V6 with clone-specific guarded-specialization feedback lines, so later LLVM runs can reuse the same per-plan suppression and retention logic instead of collapsing everything back to one function-wide specialization score.
- Taught both JIT and LLVM backends to honor multiple guarded specialization clones per function, ordered from most-specific to broadest so narrower stable-argument plans are not shadowed by broader ones.
- Extended persisted specialization planning to generate progressively broader guarded-clone candidates from the strongest measured stable values instead of only one stable-value bundle per function.
- Extended verbose JIT and LLVM reporting with specialization guard hit/fallback summaries.
- Closed the remaining live specialization-feedback parity gap by making shared/JIT/LLVM admission and optimized-entry scoring fold aggregate and clone-local guard evidence with the same precedence, so a favorable broader clone now keeps its payoff bonus even when narrower clones go neutral or unfavorable.
- Added shared `agam_profile`, JIT, and LLVM regression coverage for clone-local specialization feedback precedence so reversible specialization stays pinned across admission, retention, and emitted wrapper scoring.
- Recorded a localized specialization-payoff microbenchmark on a representative two-clone signal mix: old aggregate-only scoring took 64.13 ms across 50,000,000 updates, while the new clone-aware path took 225.04 ms across the same run, which is about 3.22 ns of extra work per score update; the current `bench_call_cache.agam` JIT run with `--call-cache` completed in 39.45 ms on the local dev binary.
- Kept the WSL LLVM path development-only via `AGAM_DEV_USE_WSL_LLVM=1`.
- Strengthened the native LLVM host-toolchain and SDK story around bundled LLVM, Visual Studio discovery, standard Windows LLVM discovery, and `agamc doctor`.
