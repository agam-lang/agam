# Phase 23: GPU and NPU Integration

## Status

- Overall status: open

## Goal

- Extend Agam into direct GPU and accelerator execution.

## Main Motive

- Let Agam target serious ML and accelerator hardware directly instead of routing everything through C++ wrappers and external frameworks.

## Responsible Crates

- `agam_codegen`
- `agam_macro`

## Planned Work

- Add an `@gpu` or `#[gpu(threads=512)]` kernel macro surface for native ML and compute kernels.
- Transition `agam_codegen` and LLVM emission toward `nvptx64` / NVPTX for CUDA and `amdgcn` / AMDGPU for ROCm.
- Add MLIR-oriented lowering and dialect planning for NPUs such as Apple Neural Engine and Qualcomm Hexagon.

## What To Build Next

- Define the kernel macro surface.
- Extend LLVM/codegen toward CUDA NVPTX and AMDGPU ROCm targets.
- Design MLIR/NPU lowering paths after the core GPU story is credible.

## Dependencies And Prerequisites

- mature LLVM backend foundation
- stable target and SDK direction
- tensor/ML native execution groundwork

## Agent Interpretation

- This is a major backend expansion phase and depends on a mature LLVM/toolchain foundation.
