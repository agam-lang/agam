# Phase 17E: First-Party Base Distributions and Official Packages

## Status

- Overall status: open

## Goal

- Ship curated first-party base distributions and define how official Agam packages live inside the organization.
- Make the default Agam install feel coherent without turning the base tool into a giant all-in-one distribution.

## Main Motive

- Users need an obvious starting point, but Agam should not bundle every SDK, package, and foreign integration by default.
- Official package governance has to be clear before the ecosystem grows across many repositories.

## Responsible Crates

- `agam_pkg`
- `agam_driver`
- `agam_std`
- `agam_ffi`

## Planned Work

- Define curated first-party package profiles such as base language support, systems/native interop, and data/AI stacks.
- Define official package naming, release, and repository governance inside the Agam organization.
- Keep foreign-language interop as explicit packs and bridges, not as the base package-manager contract.
- Align first-party distributions with the registry and environment model instead of bypassing them.

## What To Build Next

- Define the first-party profile taxonomy first.
- Then wire official package publication and install flows around that taxonomy.
- Keep the default install small and layer larger profiles through explicit environment selection.

## Dependencies And Prerequisites

- registry contract from 17C
- environments from 17D
- FFI/runtime interoperability groundwork

## Agent Interpretation

- The goal is first-party coherence, not maximum bundle size.
- Curated profiles should reduce ecosystem confusion, not hide real dependency boundaries.
