# Policy

This folder replaces the old flat policy notes that used to live at the top of `.agent/`.

## Files

- `operating-rules.md`: execution rules, verification rules, and critical directives for agents
- `project-overview.md`: what Agam is, why it exists, and the current architecture direction
- `language-design.md`: structured language-design and platform-design notes derived from the earlier bootstrapping material
- `package-ecosystem.md`: canonical package, registry, lockfile, environment, and first-party distribution architecture

## Use

- Read `operating-rules.md` when doing implementation work.
- Read `project-overview.md` when you need product and architecture context.
- Read `language-design.md` when syntax, targets, memory model, concurrency, AI/GPU, or interoperability questions come up.
- Read `package-ecosystem.md` when package management, registry layout, environments, or official package governance questions come up.
